Data Pipeline — Dual-Model, Two-Stage Flow

Generate first (streaming answer), analyse second (structured epistemic JSON). Same pipeline for free-form chat and demo scenarios.

Stage 1 — Primary Chat

POST /api/chat · Input: prompt + history · Output: text/plain stream · Model: llama-3.3-70b-versatile (fallback: 8b on rate limit)

Stage 2 — Reflect Analysis

POST /api/reflect · Input: prompt + primary_response · Output: JSON epistemic breakdown · Model: llama-3.1-8b-instant
